# リアルタイム旅行候補リスト

## 1. 概要

複数人で旅行先候補をリアルタイムに共有・編集できるWebアプリケーション。

ユーザーが旅行候補を追加・編集・削除すると、同じルームに参加している他のユーザーの画面へ即時反映される。

本アプリは将来的に開発予定の「共同編集型旅行計画アプリ」の技術検証を目的とし、以下の技術要素の習得を目的とする。

* Go
* WebSocket
* Goroutine
* Channel
* リアルタイム同期
* Room管理
* Presence（接続状態管理）
* イベント駆動設計

# 3. 画面一覧

## 3.1 ルーム一覧画面

旅行候補リストのルーム一覧を表示する。

### 機能

* ルーム作成
* ルーム一覧表示
* ルーム参加

---

## 3.2 旅行候補一覧画面

ルーム内の旅行候補を表示する。

### 表示項目

* 候補名
* メモ
* ステータス
* 作成日時

---

## 3.3 接続ユーザー表示

現在同じルームに接続しているユーザーを表示する。

例

* 田中
* 佐藤
* 山田

---

# 4. 機能要件

## 4.1 ルーム作成

### 概要

旅行候補を管理するためのルームを作成する。

### 入力項目

| 項目   | 必須 |
| ---- | -- |
| ルーム名 | ◯  |

### 成功時

ルーム一覧へ追加する。

---

## 4.2 ルーム参加

### 概要

ユーザーがルームへ参加する。

### 成功時

* WebSocket接続開始
* 接続ユーザー一覧更新

---

## 4.3 旅行候補追加

### 概要

旅行候補を追加する。

### 入力項目

| 項目  | 必須 |
| --- | -- |
| 候補名 | ◯  |
| メモ  | ×  |

### ステータス初期値

```text
未検討
```

### 成功時

同ルーム参加者へ即時通知する。

---

## 4.4 旅行候補編集

### 概要

登録済み候補を編集する。

### 編集可能項目

* 候補名
* メモ
* ステータス

### 成功時

参加者全員の画面へ即時反映する。

---

## 4.5 旅行候補削除

### 概要

登録済み候補を削除する。

### 成功時

参加者全員の画面から削除する。

---

## 4.6 ステータス変更

### 概要

候補の検討状況を管理する。

### ステータス

* 未検討
* 行きたい
* 候補
* 採用
* 却下

### 成功時

参加者全員へ即時反映する。

---

## 4.7 接続ユーザー表示

### 概要

現在同じルームに接続しているユーザーを表示する。

### 更新タイミング

* 接続時
* 切断時

---

## 4.8 編集中表示

### 概要

他ユーザーが候補を編集中であることを表示する。

### 表示例

```text
田中さんが編集中...
```

### 更新タイミング

* 編集開始
* 編集終了

---

# 5. WebSocketイベント

## CandidateCreated

候補追加通知

```json
{
  "type": "candidate_created"
}
```

---

## CandidateUpdated

候補更新通知

```json
{
  "type": "candidate_updated"
}
```

---

## CandidateDeleted

候補削除通知

```json
{
  "type": "candidate_deleted"
}
```

---

## UserJoined

参加通知

```json
{
  "type": "user_joined"
}
```

---

## UserLeft

退出通知

```json
{
  "type": "user_left"
}
```

---

## EditingStarted

編集開始通知

```json
{
  "type": "editing_started"
}
```

---

## EditingFinished

編集終了通知

```json
{
  "type": "editing_finished"
}
```

# 6. 非機能要件

## パフォーマンス

* 1ルーム最大10人程度を想定
* 候補数100件程度を想定

## リアルタイム性

* 操作後1秒以内に反映されること

## 可用性

* 個人開発用途のため高可用性は考慮しない
